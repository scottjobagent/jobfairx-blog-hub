# JobFairX Blog Playbook — 13 steps per post
**Locked:** July 21, 2026 · Goal: SEO organic traffic + LLM citations. Every post, every time.

## PLAN (before a word is written)
1. **Slot check against the link map.** Confirm the post's primary keyword from the 8-week plan, and that no existing post owns it (one keyword, one post — kills cannibalization). Add its row to the Content Inventory and its planned links (its hub + 2 siblings + pillar + a flagship stat cite) to the Link Map FIRST. The draft then follows the map.
2. **Live SERP intent check.** Google the keyword. Informational results: educational body, no pricing content, product as tactic, one soft CTA. Commercial results: sell openly. Note the check in publishing notes so it is never a guess.
3. **Research with sources.** Every stat verified at the origin, named source + year. First-party numbers only from public event pages. Every post from Week 3 on cites at least one stat from the Statistics flagship.

## DRAFT
4. **Voice:** founder-to-peer, short sentences, no AI phrases. Em dashes allowed but rare — only when the sentence truly needs one, never as a crutch. A couple per post at most; commas, periods, and colons do the work everywhere else. Answer-first lede — the hook stat in sentence one, never buried.
5. **Built for AEO:** question-form H2s, lists and tables (78% of AI Overviews contain lists), a 4-question FAQ whose answers match the schema word-for-word, quotable one-line stats.
6. **Visuals:** coded components only, never product screenshots. Max one chart. Evergreen links (calendar, never dated event pages).
7. **3-5 contextual cross-links** using the exact anchors from the Link Map.

## PACKAGE (same day as the draft)
8. **Landing card:** photo (iStock Premium plan) or brand illustration matching the site's mix, cropped to exactly 1600x900, next to the post on the hub with section assignment + download link.
9. **SEO panel on the hub:** slug, title tag, meta description, image specs (WebP, width/height, no lazy-load on first image, og:image), BlogPosting + FAQPage schema, verify-links list, and the 2-3 "TO ADD" backlink rows.
10. **Push and verify:** rendered QA locally, then checked on the live hub — zero unneeded em dashes, all links resolve, components render, card uniform. Nothing handed to Scott unchecked.

## REVIEW
11. **Scott reviews (~30 min), then Bryce's comments.** Strategy critiques get a conversation and a data check before any rewrite. The repo is canonical.

## PUBLISH (Tue/Thu)
12. **Bryce's publish day:** implement per the SEO panel, validate schema in Rich Results Test, execute the TO ADD backlink rows same day, hub page gets a sentence + link if it's a spoke, flip the post's status in the link map, request indexing in GSC.
13. **Quarterly:** stats refresh + dateModified bump — the flagship first, every benchmarks post after.

Steps 1-3 make it rank. Steps 8-9 are what Bryce executes. Step 12 is what compounds.
